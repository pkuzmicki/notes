# Test z Bezpieczeństwa Aplikacji Webowych


Wysyłasz poniższe żądanie HTTP do serwera (obrazek) za pomocą narzędzia Burp Suite, ale serwer nie zwraca żadnej odpowiedzi (status: "Waiting..."). Co jest przyczyną braku odpowiedzi serwera?

- Brakuje nagłówka Content-Type
- Metoda GET nie może zawierać nagłówka Cookie
- **Brakuje pustej linii (CRLF) po nagłówkach - serwer czeka na zakończenie żądania**
- Nagłówek Host powinien zawierać numer portu
- Parametr id=15 powinien być przesłany w ciele żądania, nie w URL


- **Brakuje pustej linii (CRLF) po nagłówkach - serwer czeka na zakończenie żądania**

## Pytanie 2 - XSS (Cross-Site Scripting)
Pentester podczas testów bezpieczeństwa aplikacji webowej wpisał w pole wyszukiwarki następującą frazę: `<u>test</u>` i zaobserwował, że tekst "test" wyświetlił się podkreślony na stronie wynikowej. Co potwierdza to zachowanie aplikacji?

- Aplikacja jest podatna na SQL Injection
- Aplikacja poprawnie enkoduje dane wejściowe użytkownika
- **W aplikacji istnieje potencjał na występowanie podatności XSS**
- Aplikacja stosuje Content Security Policy (CSP)
- Przeglądarka zablokowała próbę ataku XSS


- **W aplikacji istnieje potencjał na występowanie podatności XSS**

## Pytanie 3 - SQL Injection - komentarz
Aplikacja webowa zawiera następujące zapytanie SQL do uwierzytelniania użytkowników (obrazek). Atakujący wpisuje w pole username wartość: `admin'--`

Jakie zapytanie zostanie wykonane przez bazę danych?

- `SELECT * FROM users WHERE username = 'admin'--' AND password = '';`
- `SELECT * FROM users WHERE username = 'admin' AND password = '';`
- `SELECT * FROM users WHERE username = 'admin'-- AND password = '$password';`
- **`SELECT * FROM users WHERE username = 'admin';`**
- Zapytanie nie zostanie wykonane z powodu błędu składni

## Pytanie 4 - Błąd MySQL
Podczas testowania aplikacji webowej, po wpisaniu znaku `'` w polu wyszukiwania, serwer zwraca komunikat (obrazek). Co można wywnioskować z tej odpowiedzi?

- Aplikacja jest odporna na SQL Injection, ponieważ wykryła próbę ataku
- **Aplikacja jest podatna na SQL Injection i używa bazy MySQL**
- Zapora aplikacyjna (WAF) zablokowała żądanie
- Aplikacja używa prepared statements, które odrzuciły nieprawidłowe dane
- Serwer bazy danych jest niedostępny

## Pytanie 5 - UNION SELECT
Atakujący chce poznać strukturę bazy danych SQLite w podatnej aplikacji. Wyszukiwarka produktów akceptuje parametr q w URL. Dlaczego atakujący użył dokładnie 9 kolumn w klauzuli SELECT?

- SQLite wymaga minimum 9 kolumn w zapytaniach UNION
- **Liczba kolumn w UNION SELECT musi być identyczna z liczbą kolumn w oryginalnym zapytaniu**
- Wartość 9 to maksymalna liczba kolumn dozwolona w atakach UNION
- Jest to standardowy payload SQL Injection, który działa uniwersalnie
- Atakujący celowo zwiększył liczbę kolumn, aby uzyskać więcej danych

## Pytanie 6 - Prepared Statements
Administrator zabezpieczył formularz logowania, zamieniając dynamiczne zapytanie na prepared statement (obrazek). Dlaczego ta zmiana skutecznie chroni przed SQL Injection?

- Znak ? automatycznie filtruje znaki specjalne SQL z danych wejściowych
- **Parametry są traktowane jako dane, nie jako część struktury zapytania SQL**
- Prepared statements szyfrują dane przed wysłaniem do bazy danych
- Baza danych automatycznie odrzuca zapytania zawierające znaki specjalne
- Kod Python waliduje dane przed wykonaniem zapytania

## Pytanie 10 - Stored XSS
Administrator bloga zauważył, że po dodaniu przez użytkownika komentarza o treści `Świetny artykuł!<script>alert(document.cookie)</script>` każdy odwiedzający stronę z komentarzami widzi wyskakujące okienko z zawartością ciasteczek. Jaki typ podatności XSS został wykorzystany w tym scenariuszu?

- Reflected XSS - kod jest odbijany w odpowiedzi serwera
- DOM-based XSS - kod manipuluje Document Object Model
- **Stored XSS - złośliwy kod został zapisany w bazie danych**
- Self-XSS - użytkownik atakuje sam siebie
- Blind XSS - atakujący nie widzi bezpośrednio wyniku ataku

## Pytanie 11 - Ochrona przed XSS
Zespół programistów przygotowuje zabezpieczenia przed atakami XSS w nowej aplikacji webowej. Które z poniższych metod są skutecznymi sposobami ochrony przed XSS?

- **Używanie bibliotek typu sanitizer (np. DOMPurify) do czyszczenia HTML**
- **Stosowanie nagłówka Content-Security-Policy (CSP)**
- Wycinanie tagów HTML za pomocą wyrażenia regularnego `<.*>`
- **Enkodowanie danych w sposób odpowiedni dla kontekstu ich użycia**
- Blokowanie wszystkich żądań zawierających słowo "script"

## Pytanie 12 - XSS w atrybucie href
Aplikacja webowa pozwala użytkownikom na dodawanie linków do swoich profili. Programista zastosował enkodowanie encji HTML dla wartości atrybutu href (obrazek). Które z poniższych stwierdzeń są prawdziwe?

- Enkodowanie encji HTML w pełni chroni przed XSS w tym kontekście
- **Użytkownik może wykonać XSS poprzez podanie URL `javascript:alert(1)`**
- Atrybut href nie może być wykorzystany do wykonania kodu JavaScript
- **Należy dodatkowo walidować, czy protokół URL to HTTP lub HTTPS**
- Przeglądarka automatycznie blokuje wszystkie protokoły oprócz HTTP/HTTPS

## Pytanie 13 - XSS przez upload plików

## Pytanie 17 - SNI (Server Name Indication)
Administrator chce pobrać certyfikat SSL strony example.com za pomocą OpenSSL (obrazek). Serwer hostuje wiele domen na jednym adresie IP. Jakie ryzyko niesie użycie poniższego polecenia bez dodatkowych flag?

- Certyfikat nie zostanie zapisany do pliku
- **Serwer może zwrócić certyfikat dla innej domeny niż oczekiwana**
- Połączenie nie zostanie nawiązane z powodu błędu SSL
- Certyfikat zostanie pobrany w formacie binarnym DER zamiast PEM
- OpenSSL automatycznie wybierze najnowszy certyfikat z łańcucha

## Pytanie 19 - Deszyfrowanie TLS z ECDHE
Analityk bezpieczeństwa przechwycił ruch TLS i posiada poprawny klucz prywatny serwera. Po dodaniu klucza do Wiresharka (Edit → Preferences → Protocols → TLS → RSA keys list) ruch nadal pozostaje zaszyfrowany. Która z poniższych przyczyn jest najbardziej prawdopodobna?

- Klucz prywatny jest w złym formacie (DER zamiast PEM)
- **Połączenie używa cipher suite z ECDHE (np. TLS_ECDHE_RSA_WITH_AES_128_GCM)**
- Certyfikat serwera wygasł podczas przechwytywania
- Wireshark nie obsługuje TLS w wersji 1.2
- Plik pcap został uszkodzony podczas zapisu

## Pytanie 20 - Subject Alternative Name (SAN)
W certyfikacie X.509 znaleziono następujące rozszerzenie (rysunek). Jakie jest praktyczne znaczenie tego rozszerzenia?

- Określa listę serwerów DNS, które mogą weryfikować certyfikat
- Wskazuje alternatywne adresy IP, pod którymi dostępny jest serwer
- **Pozwala jednemu certyfikatowi chronić wiele różnych domen**
- Definiuje listę zaufanych Certificate Authority
- Określa dozwolone algorytmy szyfrowania dla tych domen

## Pytanie 21 - Deszyfrowanie TLS w Wireshark
W kontekście deszyfrowania ruchu TLS w programie Wireshark, które z poniższych stwierdzeń są prawdziwe?

- **Metoda SSLKEYLOGFILE działa dla połączeń używających ECDHE**
- Klucz prywatny serwera zawsze wystarcza do odszyfrowania każdej sesji TLS
- **Forward Secrecy uniemożliwia odszyfrowanie ruchu przy użyciu samego klucza prywatnego serwera**
- **SSLKEYLOGFILE musi być ustawiony przed nawiązaniem połączenia TLS**
- Deszyfrowanie przy użyciu klucza prywatnego działa dla wszystkich wersji TLS

## Pytanie 22 - Błąd certyfikatu
Rozważmy następujący scenariusz: przeglądarka wyświetla ostrzeżenie "NET::ERR_CERT_AUTHORITY_INVALID" podczas próby połączenia z serwerem HTTPS. Które z poniższych sytuacji mogą być przyczyną tego błędu?

- Serwer używa zbyt silnego algorytmu szyfrowania

## Pytanie 25 - Burp Suite Intruder
W narzędziu Burp Suite Intruder chcesz przeprowadzić atak bruteforce na formularz logowania, testując wszystkie kombinacje loginów (100 wartości) i haseł (500 wartości). Który typ ataku powinieneś wybrać?

- Sniper - testuje jeden parametr na raz z jednej listy

W aplikacji DVWA na poziomie zabezpieczeń MEDIUM próba ataku XSS za pomocą `<script>alert('XSS')</script>` nie działa, ponieważ słowo 'script' jest usuwane z danych wejściowych. Który z poniższych payloadów pozwoli na obejście tego zabezpieczenia?

✅ **`<SCRIPT>alert('XSS')</SCRIPT>` - zmiana wielkości liter**

## Pytanie - XSS przez onerror
Aplikacja webowa generuje dynamicznie kod HTML na podstawie parametru URL. Programista zastosował enkodowanie encji HTML dla wartości atrybutu src (obrazek). Które z poniższych stwierdzeń są prawdziwe?

✅ **Atak XSS jest możliwy poprzez użycie spacji i dodanie atrybutu 'onerror'**
✅ **Brak cudzysłowów wokół wartości atrybutu src stanowi lukę bezpieczeństwa**

## Pytanie - OpenSSL polecenia
Administrator systemu chce wyodrębnić informacje z certyfikatu TLS. Które z poniższych poleceń OpenSSL poprawnie wyświetlą żądane informacje?

✅ **`openssl x509 -in cert.pem -noout -dates` - wyświetla daty ważności certyfikatu**
✅ **`openssl x509 -in cert.pem -noout -issuer` - wyświetla informację o wystawcy (CA)**

## Pytanie - URL encoding
Chcesz przesłać w parametrze URL następujący tekst: `name=Jan Kowalski&age=25`. Które z poniższych zakodowanych wartości są poprawne dla kodowania procentowego (URL encoding)?

✅ **`name=Jan+Kowalski%26age=25`**
✅ **`name=Jan%20Kowalski%26age%3D3025`**
